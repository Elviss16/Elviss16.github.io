REST API integration using HTML, Bootstrap, and vanilla JavaScript.

Overview

This project is a small Formula 1 dashboard that connects to the OpenF1 REST API. It shows a sample list of drivers and basic telemetry (speed, RPM, gear, etc.) for a selected driver. Everything is built with plain HTML, Bootstrap for styling, and JavaScript's fetch() for API calls.

API Used

  • Name: OpenF1
  • Base URL: https://api.openf1.org/v1
  • Authentication: No API key required
  • Response format: JSON

Note: Calling the base URL by itself will return a JSON error saying the route is invalid. The API requires a valid endpoint such as /drivers or /car_data.

Endpoints Used in This App

1. /drivers

Used on the home page and telemetry page to load driver information (full name, country code, team name, and car number).

Example request:

https://api.openf1.org/v1/drivers?driver_number=16

2. /car_data

Used on the telemetry page to load basic telemetry for a selected driver in a sample session.

Example request:

https://api.openf1.org/v1/car_data?driver_number=16&session_key=9159

How the Application Works

  • On index.html, JavaScript calls /drivers and creates Bootstrap cards for a sample of drivers. Each card is colored using a simple team-color mapping.
  • When the user clicks View Telemetry, they are sent to telemetry.html?driver=XX.
  • On telemetry.html, the app reads the driver number from the URL and:
    • calls /drivers?driver_number=XX to show the driver info;
    • calls /car_data?driver_number=XX&session_key=9159 and filters for rows where speed is higher than 150 km/h.
  • If the API does not return any high-speed data for that driver, the page shows a friendly error message instead of an empty table.

Libraries / Tools

  • Vanilla JavaScript fetch() for making REST calls
